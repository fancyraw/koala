## 一、需求背景
一款个体户商城微信小程序

## 二、目标
一句话：C端用户浏览商品、加购、领券、下单、微信支付、查看订单，后台管理员商品管理、券管理、订单管理、用户管理、数据看板、运维管理


## 三、产品需求

### 1. 页面模块
- C端页面设计.md
- 后台管理页面设计.md

### 2. 商品
- 商品字段：名称、主图、原价、售价、规格（最多4个）、标签（热销/新品/礼盒/特惠）、是否推荐、库存、分类、详情图、产品亮点
- 划线价为原价/市场参考价，现价为售价，券为额外优惠，三者独立
- 首页热销爆款：按销量排序取 Top N
- 首页精选推荐：后台标记为「推荐」的商品，无推荐商品时按上架时间倒序展示全部

### 3. 分类
- 后台可配置分类名称、图标、排序
- 同步到C端首页品类专区和商品列表页筛选栏
- 点击品类传递分类ID，列表页按分类过滤

### 4. 优惠券
两种券：满减券（满100减10）、无门槛券（直减5元）
- 后台创建填写：券名、券类型、优惠金额（满减券额外填门槛金额）、有效期、每人限领张数、发行总量
- 领取入口：首页领券横条、商品详情页价格区下方，用完可再领
- 发券场景：新人注册自动发券、分享链接领券、后台手动发券
- 结算时系统自动使用最优券，一单只能用一张
- 券状态：未开始/进行中/已结束/已停发

### 5. 订单
- 用户链路：加购 → 购物车 → 确认订单（选地址/选券）→ 微信支付 → 支付结果 → 订单详情
- 订单状态：待付款 → 待发货 → 配送中 → 已完成
- 待付款订单超过30分钟未支付自动取消
- 退款：待发货状态可申请退款，后台处理

### 6. 配送
- 基础运费后台可配
- 满额包邮门槛后台可配

### 7. 内容管理
**Banner**
- 后台配置图片、跳转链接、标签文字、排序、上下线状态
- 同步到C端首页Banner轮播区

**品牌信息**
- 后台配置品牌名称、App图标、Logo、品牌故事（标题+封面图+纯文本简介200字以内）
- 品牌名称同步到C端顶部导航和页面标题
- 品牌故事同步到C端个人中心品牌故事卡片
