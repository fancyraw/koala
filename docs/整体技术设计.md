## 四、关键模块设计

### 4.1 用户登录（C端）

```
wx.login() 取 code → /api/user/login
  → code 换 openid → 查 user_auth(wechat_mp, openid)
    存在：登录；不存在：事务创建 user + user_auth（自动注册），标记 newbie_coupon_status=0
  → 发 JWT
新人券采用「待领取」模式，不在注册时自动入账（见 4.9）
```

### 4.2 管理员登录（后台）
微信扫码（主）/ 手机号+验证码（备），匹配 admin 表，发 JWT；role 控制运维管理可见性。

### 4.3 价格计算（核心，与产品算价口径一致）

**固定优先级，逐级扣减**（金额均不为负）：

```
1. 商品合计 = Σ(sku.price × qty)
2. 会员折扣 = 商品合计 × (1 - rate)        rate: 普通1 / 白银0.98 / 黄金0.95
   会员价小计 = 商品合计 - 会员折扣
3. 优惠券抵扣（无门槛 + 满减 各一张叠加）
   - 满减门槛以「商品合计(原价)」判断
   - 券后小计 = 会员价小计 - (无门槛面额 + 满减面额)，下限 0
4. 积分抵扣 = min(可用积分可兑额, 券后小计 × 50%)   100积分=1元
   - 仅当可抵金额 ≥ ¥0.1 时确认订单页默认开启
   - 抵扣后 = 券后小计 - 积分抵扣
5. 运费：满额包邮门槛以「商品合计」判断；运费不参与折扣/积分
6. 实付 = 商品合计 - 会员折扣 - 券抵扣 - 积分抵扣 + 运费（最低 0.01）
本单可得积分 points_award = floor( 实付商品金额 × 会员倍率 )，确认收货后发放
```

`/api/order/preview` 与 `/api/order/submit` 共用同一算价服务，保证预览与下单一致。

**商详页价格与优惠（静态，与购物车解耦）**：`/api/products/{id}` 返回——**售价** + **会员折扣金额**（售价 ×（1 − 会员折扣率））+ **券省**（无门槛券取最大 + 满足门槛的满减券取最大，以「商品售价 × 1」判断门槛、不满足不计入）。前端展示「售价」+ 一行「**下单省 ¥X = 会员 ¥Y + 券 ¥Z**」优惠明细。**首页商品卡只展示售价 + 已售**（不放会员价/券省，避免密集）。**不读购物车**，未登录返回开通会员 / 新人券引导，最终以购物车 / 下单算价为准。

### 4.4 下单 + 资产冻结（提交订单先锁定，再拉起支付）

```
提交订单（单事务，加分布式锁 user 维度防并发）：
  a. 幂等校验：前端提交令牌(submitToken) 在 Redis SETNX，重复提交直接拦截
  b. 校验 + 扣库存（乐观锁 UPDATE stock=stock-N WHERE stock>=N）
  c. 算价（4.3），落 order 各金额字段
  d. 【锁定券】user_coupon status 0→3(已锁定)，lock_order_no=订单号；写 order_coupon
  e. 【冻结积分】user.points -= points_used；写 point_record(type=2 下单抵扣)
  f. 创建 order(status=0) + order_item
  g. expire_at = now + 30min；返回订单号
  → 调微信统一下单，前端 wx.requestPayment()
```

### 4.5 支付结果处理（释放 / 核销，最高优先级）

```
支付成功回调 /api/order/wx-notify（幂等：transaction_id 唯一 + 状态 CAS）：
  - order status 0→1(待发货)（CAS，已处理则直接返回成功）
  - user_coupon 3→1(已使用)、coupon.used_count+1
  - 积分抵扣转正式（已扣，无需再动），payment 置成功
  - user.total_spent += 实付；重算 member_level（升级判定）

支付失败 / 用户取消 / 超时未支付（30min 定时任务）→ 释放资产（单事务）：
  - 回滚库存（stock += N）
  - 释放券：user_coupon 3→0(未使用)
  - 返还积分：user.points += points_used；写 point_record(type=3 抵扣返还)
  - order status →4(已取消)
⚠ 资产释放必须与支付失败/取消回调同事务、最高优先级，避免「券/积分没了」投诉
```

### 4.6 确认收货（幂等 + 发积分 + 会员升级）

```
/api/orders/{no}/confirm （前端按钮点击即置灰+loading，防重复提交）：
  - 状态机 CAS：UPDATE order SET status=3 WHERE order_no=? AND status=2
    影响行数=0 → 已处理，幂等返回（防重复点击/网络重试）
  - 发放积分：user.points += points_award；写 point_record(type=1 消费获得, expire_at=now+12月)
  - completed_at=now
（total_spent 与会员升级在支付成功时已处理；积分确认收货才发，规避先用后退）
```

### 4.7 售后 / 退款流程

```
状态机：0待审核 → 1审核通过待寄回 →(退货退款)2买家已寄回 → 3商家已收货 → 4已退款
                                  └(仅退款,未发货)──────────────────→ 4已退款
                       └ 5已拒绝

仅退款（未发货,待发货订单）：商家审核通过 → 直接原路退款 → 4
退货退款（已发货,待收货订单）：审核通过 → 买家寄回(填 return_tracking_no) → 商家确认收货 → 退款 → 4

退款资金与权益（退款时统一处理，事务）：
  - 现金原路退回（调微信退款），payment→3
  - 优惠券：未过期则原路退回 user_coupon→0
  - 积分：返还下单抵扣的积分(type=3)
  - 若订单已完成(积分已发放)再退货退款：回收已发积分(type=5)；
    余额不足则允许 user.points 为负，或从退款金额按 100积分=1元 抵扣等额现金后再退
  - 回滚销量/库存（按业务策略）
```

### 4.8 物流查询

```
发货：后台填 logistics_company + logistics_no → order.shipped_at, status→2
C端查看物流：/api/orders/{no}/logistics
  → 后端调 快递100/快递鸟 查询 API（按公司+单号），结果缓存 Redis(如30min)
  → 返回轨迹节点列表(描述+时间)，最新在前
  → C端弹窗展示：节点>5 默认 3-4 条 + 全屏物流页查看全部；待发货无入口
```

### 4.9 新人券发放与异常兜底

```
注册不自动入账，user.newbie_coupon_status=0(待领取)
首页判断：若 status=0 且券进行中 → 弹新人券弹窗
点「立即领取」→ /api/coupons/claim-newbie：
  - 成功：发 user_coupon，user.newbie_coupon_status=1，coupon.claimed_count+1
  - 失败：返回错误，前端弹窗原地提示「网络开小差了，再试一次」，不关闭
未领就关闭：status 仍为 0 → 下次进首页继续弹，直至领取成功或券失效(置2)
并发/重复领取：claim 接口加用户级幂等 + per_user_limit 校验，防超发
```

### 4.10 幂等与防重设计（统一约定）

| 场景 | 方案 |
|------|------|
| 提交订单 | 前端 submitToken（一次性）+ Redis SETNX 拦截重复；前端按钮点击即置灰 loading |
| 确认收货 | 前端按钮置灰 loading + 后端状态机 CAS（status=2→3，影响 0 行视为已处理） |
| 微信支付回调 | transaction_id 唯一键 + 订单状态 CAS，重复回调直接返回成功 |
| 申请退款 / 售后 | order_no 维度校验当前状态，重复申请拦截 |
| 领新人券 | 用户级幂等 + per_user_limit |
| 通用写接口 | 核心操作走数据库唯一约束 / 乐观锁，避免依赖前端 |

### 4.11 图片上传
后台直传七牛云：/api/admin/upload/token 取 Token → 前端直传 → 返回 CDN URL 入库。

### 4.12 操作日志
AOP 拦截 POST/PUT/DELETE，异步写 operation_log（操作人/模块/动作/内容/IP）。含 C 端（登录/地址/购物车/领券/订单）、后台（商品/订单/券/用户/配置/Banner/SQL）、系统（超时取消/券过期/积分过期/备份）。

### 4.13 数据库定时备份
每天凌晨 3:00 mysqldump 全库 → gzip → 上传七牛云备份 Bucket，保留 30 天，失败记日志。

## 六、部署架构

### 6.1 Docker Compose

```yaml
services:
  mysql:  { image: mysql:5.7, ports: ["3306:3306"], volumes: ["./data/mysql:/var/lib/mysql"] }
  redis:  { image: redis:7-alpine, ports: ["6379:6379"] }
  server: { build: ./server, ports: ["8080:8080"], depends_on: [mysql, redis] }
  web:    { build: ./web }          # Nginx 托管后台 Vue 构建产物
  nginx:  { image: nginx:alpine, ports: ["80:80","443:443"], depends_on: [server, web] }
```

### 6.2 Nginx 路由
```
/        → 后台管理 Web
/api/    → 后端 API(:8080)
```
C端小程序经微信审核后访问 `https://域名/api/`。

### 6.3 关键定时任务

| 任务 | 频率 | 说明 |
|------|------|------|
| 订单超时取消 | 每 1 分钟 | 扫 status=0 且 expire_at<now，**释放库存/券/积分** |
| 优惠券过期 | 每天凌晨 | user_coupon 到期 → 状态 2 |
| 积分过期 | 每天凌晨 | point_record 到期积分扣减并写流水(type=6) |
| 数据库备份 | 每天 3:00 | mysqldump → gzip → 七牛云，保留 30 天 |
```
