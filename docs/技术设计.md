## 一、整体架构

### 1.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| C端 | uni-app（Vue3） | 编译为微信小程序 |
| 后台管理 | Vue3 + Element Plus | SPA，Nginx 托管静态资源 |
| 后端 | Spring Boot 2 + JDK 8 | RESTful API |
| 数据库 | MySQL 5.7 | 核心业务数据 |
| 缓存 | Redis | 会话管理、购物车、验证码、热数据缓存 |
| 文件存储 | 七牛云（主）+ Cloudflare（备） | 商品图、Banner图、品牌图等 |
| 支付 | 微信支付（JSAPI） | 小程序内调起支付 |
| 登录 | 微信授权（C端）/ 微信扫码（后台） | 后台备选手机号+验证码登录 |
| 部署 | Docker + Docker Compose | 腾讯云服务器 |
| 反向代理 | Nginx | HTTPS、静态资源、API转发 |

## 三、数据表设计

### 3.1 用户相关

```sql
-- 用户表（不存登录凭证，登录凭证在 user_auth 表）
CREATE TABLE user (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    nickname        VARCHAR(64) DEFAULT '' COMMENT '昵称',
    avatar_url      VARCHAR(512) DEFAULT '' COMMENT '头像URL',
    phone           VARCHAR(20) DEFAULT '' COMMENT '手机号',
    total_spent     DECIMAL(10,2) DEFAULT 0.00 COMMENT '累计消费金额',
    status          TINYINT DEFAULT 1 COMMENT '1=正常 0=禁用',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_phone (phone)
) COMMENT='用户表';

-- 用户登录凭证表（支持多端登录扩展）
CREATE TABLE user_auth (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT NOT NULL COMMENT '关联 user.id',
    auth_type       VARCHAR(16) NOT NULL COMMENT '凭证类型：wechat_mp/phone/email',
    auth_id         VARCHAR(128) NOT NULL COMMENT '凭证标识：openid/手机号/邮箱',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_type_id (auth_type, auth_id),
    INDEX idx_user_id (user_id)
) COMMENT='用户登录凭证表，MVP阶段只有 wechat_mp 一种类型';

-- 用户地址表（通用结构，支持小程序/H5多端）
CREATE TABLE user_address (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT NOT NULL,
    name            VARCHAR(32) NOT NULL COMMENT '收货人姓名',
    phone           VARCHAR(20) NOT NULL COMMENT '手机号',
    province        VARCHAR(32) NOT NULL COMMENT '省',
    city            VARCHAR(32) NOT NULL COMMENT '市',
    district        VARCHAR(32) NOT NULL COMMENT '区',
    detail          VARCHAR(256) NOT NULL COMMENT '详细地址（街道门牌号）',
    full_address    VARCHAR(512) NOT NULL COMMENT '完整地址（冗余，展示用）',
    is_default      TINYINT DEFAULT 0 COMMENT '是否默认地址',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id)
) COMMENT='用户地址表';
-- 地址录入方式（用户可选）：
--   方式一：wx.chooseAddress() 一键导入微信地址簿
--   方式二：三级联动选择器手动填写（china-area-data npm 包，纯前端，无需后端 region 表）
-- 两种方式统一存入相同字段结构，H5端复用方式二
```

## 四、关键模块设计

### 4.1 用户登录（C端）

```
用户打开小程序 → wx.login() 获取 code
    → 前端携带 code 调后端 /api/user/login
    → 后端用 code 换 openid（调微信接口）
    → 查 user_auth 表（auth_type='wechat_mp', auth_id=openid）
      → 存在：取 user_id 查 user 表，登录成功
      → 不存在：创建 user + user_auth 记录（事务），自动注册
    → 生成 JWT Token 返回前端
    → 新注册用户自动发放新人券（异步）

未来扩展（非 MVP）：
    手机号登录 → 查 user_auth（auth_type='phone', auth_id=手机号）
    邮箱登录   → 查 user_auth（auth_type='email', auth_id=邮箱）
    绑定多登录方式 → 同一 user_id 下新增 user_auth 记录
```

### 4.2 管理员登录（后台）

```
方案一：微信扫码登录
    管理员打开后台 → 展示微信登录二维码
    → 扫码后回调获取 openid
    → 匹配 admin 表 wx_openid → 登录成功，发 JWT

方案二：手机号+验证码（备选）
    管理员输入手机号 → 获取短信验证码
    → 验证通过 → 匹配 admin 表 phone → 登录成功，发 JWT
```

### 4.3 下单流程

```
1. 确认订单页加载
   └ 前端传购物车选中项（sku_id + quantity 列表）
   └ 后端校验库存、计算商品总价、查可用券，返回预览数据

2. 提交订单
   └ 前端传：地址ID、商品列表、选用的coupon_id、备注
   └ 后端：
     a. 再次校验库存 + 扣减库存（乐观锁：UPDATE stock = stock - N WHERE stock >= N）
     b. 计算金额：商品总价 + 运费 - 优惠券 = 实付
     c. 创建订单 + 订单项（事务）
     d. 标记优惠券已使用
     e. 清除购物车已下单商品
     f. 设置 expire_at = now + 30分钟
     g. 返回订单号 + 预支付参数

3. 微信支付
   └ 后端调微信统一下单接口，获取 prepay_id
   └ 前端 wx.requestPayment() 调起支付
   └ 微信支付回调通知后端 → 更新订单状态为待发货 → 累加用户消费额

4. 超时取消（定时任务）
   └ 每分钟扫描 status=0 AND expire_at < now 的订单
   └ 回滚库存、退还优惠券、更新状态为已取消
```

### 4.4 优惠券匹配逻辑

```
用户进入确认订单页时：
1. 查 user_coupon WHERE user_id=? AND status=0（未使用）
2. JOIN coupon 表获取券规则
3. 过滤：当前时间在有效期内 AND coupon.status=1（未停发）
4. 过滤：订单商品总价 >= min_spend（满减门槛）
5. 按 discount_amount DESC 排序
6. 默认选中第一张（抵扣最大的）
7. 不满足门槛的券返回但标记为不可用
```

### 4.5 购物车

```
存储：MySQL cart_item 表（登录用户持久化）
加购：UPSERT（存在则 quantity+N，不存在则 INSERT）
修改数量：直接 UPDATE quantity
删除：DELETE
结算后清除：DELETE 已下单的 cart_item
```

### 4.6 图片上传

```
后台上传图片（商品图/Banner图/品牌图等）：
1. 前端选择文件 → 调后端 /api/admin/upload
2. 后端生成七牛云上传 Token + 文件名（按日期+UUID命名）
3. 前端直传七牛云（减轻服务端带宽压力）
4. 上传成功后返回 CDN URL 存入数据库
```

### 4.7 操作日志

```
全端所有写操作自动记录到 operation_log 表，用于问题回溯和审计：
1. 通过 AOP 切面拦截所有 POST/PUT/DELETE 请求
2. 自动提取：操作人类型+ID（从JWT）、模块（从URL路径）、动作（从HTTP方法）、内容（从请求参数）
3. 异步写入日志表，不影响主业务性能
4. 运维管理页面可查看全部操作日志，支持按来源/模块/操作人/时间筛选

C端用户操作记录：
  - 登录/注册
  - 地址：新增/编辑/删除
  - 购物车：加购/修改数量/删除
  - 优惠券：领取
  - 订单：提交/支付/取消/确认收货/申请退款

后台管理员操作记录：
  - 商品：新增/编辑/上下架/删除
  - 订单：发货/处理退款
  - 优惠券：新建/编辑/停发
  - 用户：禁用/启用
  - 配置：修改配送设置/支付设置/品牌信息
  - Banner：新增/编辑/上下线/删除/排序
  - 运维SQL：执行的SQL语句和影响行数

系统自动操作记录：
  - 订单超时自动取消
  - 优惠券过期批量处理
```

### 4.8 数据库定时备份

```
每天凌晨自动备份数据库，上传到七牛云：
1. 定时任务触发（每天凌晨3:00）
2. 执行 mysqldump 导出全库 SQL 文件
3. gzip 压缩（文件名：koala_db_2026-06-10_030000.sql.gz）
4. 调用七牛云 SDK 上传到专用备份 Bucket
5. 上传成功后删除本地临时文件
6. 记录操作日志（operator_type=3 系统）

备份策略：
  - 频率：每天一次（凌晨3:00，业务低峰期）
  - 保留：七牛云保留最近30天备份，超期自动删除（通过七牛生命周期规则）
  - 备份文件路径：backup/db/koala_db_{yyyy-MM-dd_HHmmss}.sql.gz
  - 失败处理：备份失败时记录错误日志，可在运维管理页面查看
```

## 六、部署架构

### 6.1 Docker Compose

```yaml
services:
  mysql:
    image: mysql:5.7
    ports: ["3306:3306"]
    volumes: ["./data/mysql:/var/lib/mysql"]

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]

  server:
    build: ./server
    ports: ["8080:8080"]
    depends_on: [mysql, redis]

  web:
    build: ./web
    # Nginx 托管 Vue 构建产物

  nginx:
    image: nginx:alpine
    ports: ["80:80", "443:443"]
    volumes: ["./nginx.conf:/etc/nginx/nginx.conf"]
    depends_on: [server, web]
```

### 6.2 Nginx 路由

```
/              → 后台管理 Web（web 容器）
/api/          → 后端 API（server 容器 :8080）
```

C端小程序通过微信审核后直接访问 `https://域名/api/` 。

### 6.3 关键定时任务

| 任务 | 频率 | 说明 |
|------|------|------|
| 订单超时取消 | 每1分钟 | 扫描待付款且已过期的订单，回滚库存/退还优惠券 |
| 优惠券过期处理 | 每天凌晨 | 将过期的 user_coupon 状态改为已过期 |
| 数据库备份 | 每天凌晨3:00 | mysqldump 全库导出 → gzip 压缩 → 上传七牛云，保留30天 |
