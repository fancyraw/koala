## C端页面 UI 设计说明

> 视觉基调：**小红书 App 风格** —— 轻盈通透、极致留白、图片即主角、大圆角柔和、UI 弱化让位于内容。红色只作点睛，整体由黑白灰构成高级感。

### 全局设计规范（Design Tokens）

#### 色彩

- **品牌红** `#FF2442`：小红书标志红。仅用于价格、强 CTA、角标、促销标签，是"点睛色"，不大面积使用
- **品牌红浅底** `#FFECEC`：标签底色（浅红底 + 红字），如「新品」「直降」
- **会员金** 渐变 `#E8C97A → #C9A24B`：会员等级徽章、会员卡，区别于品牌红，体现"尊享"质感
- **主标题** `#1A1A1A`：商品名、页面标题、价格数字
- **正文** `#333333`：常规文字
- **次要文字** `#999999`：副信息、辅助说明
- **占位/弱文字** `#C8C8C8`：placeholder、未选中态
- **页面背景** `#F8F8F8`：浅灰背景，让白色卡片"浮起来"，是小红书精致感的关键
- **卡片/内容底** `#FFFFFF`：所有卡片纯白
- **分隔/描边** `#F0F0F0`：极浅分割线，能不用就不用，靠留白区分

#### 字体

- **大标题** 18px / 600：页面主标题、商品详情名
- **标题** 15-16px / 600：区块标题、卡片商品名
- **正文** 13-14px / 400：常规内容
- **辅助** 11-12px / 400：已售、副信息
- **价格** 数字 16-26px / 700，¥ 符号缩小到 0.6em：小红书式价格：符号小、数字大、紧凑
- **字体族** PingFang SC / SF Pro，全程无衬线，绝不用手写体

#### 形状与质感

- **卡片圆角**：**16px**（大而柔，小红书核心特征），小元素 8-12px
- **图片圆角**：商品图 12px；瀑布流/推荐卡片图片为顶部通图，圆角 16px 16px 0 0
- **阴影**：几乎不用硬阴影。卡片靠 `#F8F8F8` 背景与白卡对比浮起；必要时极轻 `0 1px 4px rgba(0,0,0,.04)`
- **按钮**：全圆角胶囊（border-radius 999px）。主按钮：品牌红 `#FF2442` 纯色实心 + 白字 + 600；次按钮：`#F5F5F5` 浅灰底 + `#333` 文字
- **标签 Pill**：浅色底 + 同色系文字，圆角 6px，小字 10-11px，内边距紧凑（2px 6px）
- **间距**：基础 8px。模块之间用 **8px 浅灰留白带** 分隔（而非分割线），左右安全边距 12-16px
- **图标**：极简线性，2px 描边，风格统一；激活态填充

#### 关键氛围词
留白充足、图片大占比、卡片轻浮、红色克制、信息精炼、可"逛"的种草感。

#### 全局反馈组件

- **Toast 轻提示**：**全局统一**的弱反馈：屏幕居中（或顶部）深色半透明圆角胶囊 `rgba(0,0,0,.75)` + 白字 13px，1.5-2s 自动消失，不打断操作。用于「网络开小差了，再试一次」「已加入购物车」「已达库存上限」「复制成功」等通用反馈，保证全局异常 / 操作反馈体感一致（不再各处弹窗内单独实现）
- **Loading**：按钮内联 loading（提交 / 确认收货点击后置灰 + 转圈）；整页 loading 用于支付确认中等中间态
- **整页加载失败**：页面级数据请求失败时展示：居中线性断网/错误图标 80px `#E0E0E0` + 「加载失败，请稍后重试」13px `#999` + 下方「重新加载」品牌红文字按钮 14px。点击重新请求数据
- **下拉刷新**：所有列表页（商品列表、订单列表等）支持下拉刷新：下拉时顶部出现品牌红旋转圈 + 「刷新中…」小字，松手触发刷新，完成后自动收起
- **加载更多**：长列表底部触底自动加载下一页：加载中展示灰色旋转圈 + 「加载中…」11px `#999`；无更多数据展示「— 没有更多了 —」11px `#C8C8C8`

#### 未登录态

- **操作拦截**：未登录时点击加购/购买/购物车Tab/个人中心Tab，**弹出底部微信授权登录浮层**（白底圆角 20px 上滑）：品牌 Logo + 「登录后享受更多权益」+ 「微信一键登录」品牌红实心大胶囊 + 底部「暂不登录」灰色文字。登录成功后自动继续原操作
- **个人中心**：会员头部卡替换为浅灰渐变卡：默认头像圆形 56px `#E0E0E0` + 「点击登录」18px 600 `#333` + 下方「登录享会员权益」12px `#999`。功能菜单中「我的优惠券」数字显示「--」`#C8C8C8`，订单区域隐藏

#### 商品卡片模板（通用）

> 商品卡片在**首页热销爆款、首页精选推荐、商品列表瀑布流**三处复用，统一规范如下，避免各处样式漂移。三种形态共用同一信息结构与交互。

**共用信息结构（精简版）**：商品图 + 商品名 + 价格 + **加购按钮**。
- **不展示优惠券标签**（券优惠统一在购物车/结算页体现，卡片只露价格，保持干净种草感）
- **商品标签角标**：商品图**左上角**展示后台配置的**单个**商品标签（热销/新品/礼盒/特惠），红标贴角样式（`#FF2442` 实心 + 白字 9px，圆角 4px）。**后台未配标签则不展示**；三种形态位置一致。**不再有按销量自动生成的「爆款」角标**
- **不展示「已售 N」**，原「已售 N」的位置改为**加购按钮**
- **价格**：`¥`12-14px 红 + 数字 15-18px 红 700，无划线价

**加购按钮（统一规范）**：价格行右侧 **28px 圆形品牌红 `#FF2442` 实心按钮 + 白色「+」线性图标**。
- 点击行为：**单规格商品直接加入购物车**；**多规格商品弹出底部「选择规格」浮层**（选规格 + 数量后加入）。加入成功后购物车角标 +1，并轻动效 / Toast「已加入购物车」
- **售罄态**：按钮置灰 `#E0E0E0` 不可点
- 加购按钮为独立点击热区；**点击卡片其余区域进商品详情页**

**三种形态**：
- **形态 A · 竖卡（热销爆款）**：卡片宽 130px 圆角 16px 白底，横向滑动露出 2.4 张。顶部 1:1 通图 + 左上角商品标签红标贴角（后台配置，无则不展示）；信息区 padding 10px：商品名 13px 单行省略 + 价格行（左价格 / 右加购按钮）
- **形态 B · 左右图文卡（精选推荐）**：卡片圆角 16px 白底，横向 flex。左侧方图（1:1，宽约 120px，圆角 12px）；右侧信息区 padding 12px 14px：商品名 15px 600 单行 + 种草语（＝复用商品「产品亮点」首条，无亮点则隐藏该行）2 行 13px `#666` + 底部价格行（左价格 17px / 右加购按钮）
- **形态 C · 瀑布流卡（商品列表）**：双列，卡片圆角 16px 白底。顶部商品图（高度不等制造瀑布感，圆角 16px 16px 0 0）；信息区 padding 10px：商品名 13px 最多两行 + 价格行（左价格 15px / 右加购按钮）。**售罄态**：商品图叠加半透明蒙层 + 居中「已售罄」白字 12px 600，价格行与加购按钮转灰

#### 首页

```
┌──────────────────────────────────────┐
│ 顶部：Logo + 品牌名               🛒   │
├──────────────────────────────────────┤
│ Banner 轮播区（纯图，无标签）            │
├──────────────────────────────────────┤
│ 品类专区（图标网格）                      │
├──────────────────────────────────────┤
│ 热销爆款（横向滑动卡片列表 + 查看全部）   │
├──────────────────────────────────────┤
│ 精选推荐（左右卡：图左文右屏效高）          │
└──────────────────────────────────────┘

﹝叠加﹞领券弹窗：系统下发券后，前端查询到有新到账券时弹窗提示
```

- **页面背景**：`#F8F8F8` 浅灰，所有内容区为白色，模块间 8px 灰留白带
- **顶部导航**：高度 48px，白色背景。左侧品牌 Logo（圆角方形 28px，红色）+ 紧跟品牌名 `#1A1A1A` 17px 600；**右上角放购物车入口**（28px 线性购物车图标 `#1A1A1A`，右上角带品牌红件数角标，点击进购物车页）。不放搜索框和消息图标，保持轻盈
- **Banner 轮播区**：白色容器内，左右边距 12px，圆角 16px。**纯图片轮播，不叠加任何标签角标**，让主视觉干净。底部居中指示器：当前页白色长条（宽 14px 高 3px 圆角），其余白色半透明小圆点。点击跳转商品列表/活动页
- **品类专区**：白色容器。一行 5 个品类均分：每个为 **48px 圆角方形**（圆角 14px）浅彩底（每类不同的极浅马卡龙色），内置线性图标，下方品类名 11px `#333`，间距 6px
- **热销爆款**：标题「🔥 热销爆款」`#1A1A1A` 16px 600 + 右「查看全部 ›」`#999` 13px。横向滑动卡片，露出 2.4 张。卡片采用**商品卡片模板·形态 A（竖卡）**：顶部 1:1 通图 + 左上角商品标签红标贴角（后台配置，无则不展示）；商品名 + 价格行（左价格 / 右加购按钮）。点击卡片进详情、点加购按钮直接加购
- **精选推荐（左右图文卡）**：**首页核心种草位**。标题「✨ 为你推荐」。采用**商品卡片模板·形态 B（左右图文卡）**：左方图 + 右信息（商品名 + 2 行种草语＝复用「产品亮点」首条 + 底部价格行：左价格 / 右加购按钮）。卡片间距 12px，左图右文紧凑排布，一屏展示更多商品，屏效更高
- **领券弹窗（系统下发，前端查到即弹）**：**券由系统/后台下发到用户账户**（新人券按「新人」条件下发、活动券按各自条件下发）。前端在登录 / 进首页时**查询账户是否有新到账、未告知的券**，**查到则弹窗提示**（无新券则不弹）。居中模态弹窗（宽约 300px，圆角 20px）：顶部红→金渐变区 + 大字「🎁 已为你发放优惠券」+ 副文案「下单立减」13px。中部**券列表（支持多张）**：每张一行——左金额「¥5」「¥10」大号 + 右券名与门槛「新人券 · 无门槛」「满100减10」+ 有效期 11px `#999`。底部「**去使用**」品牌红实心大胶囊（券已在账户，引导购物，**非「立即领取」**）。右上 / 底部关闭 ✕，半透明黑遮罩。**异常兜底**：查询失败则本次不弹、下次再查；用户未操作关闭则下次进首页若仍有未告知券会再次提示

- **商品名称**：`#1A1A1A` 17px 600，行高 24px，最多两行
- **规格选择**：白卡。标题「规格」15px 600。规格 pill 横排：未选 `#F5F5F5` 底 `#333`，选中 `#FFECEC` 浅红底 + `#FF2442` 红字 + 红描边 1px。圆角 8px padding 8px 16px。**单规格商品**：若该商品只有一个规格，规格区可不展示（或展示但默认选中、不可切换）
- **数量步进**：行内：左「数量」15px，右步进器「− 1 +」，按钮 28px 圆形 `#F5F5F5` 底。**禁用态**：① 数量 = 1 时「−」按钮降透明度（`opacity .35`）且不可点；② 数量达**库存上限**时「+」按钮置灰不可点，并 Toast「已达库存上限」；③ **单次下单达到限购数量**时「+」同样置灰不可点，并 Toast「该商品每单限购 N 件」

- **底部操作栏**：白底高 56px+安全区。左侧「客服」竖排小图标（44px，图标+8px小字 `#999`），右侧两个胶囊按钮：「加入购物车」浅红 `#FFECEC` 底红字；「立即购买」品牌红 `#FF2442` 实心白字。两按钮高 40px 全圆角。**不含收藏图标**
- **售罄态**：库存为 0 时：① 商品主图左下角叠加半透明黑底白字「已售罄」标签（圆角 6px，`rgba(0,0,0,.55)` + 白字 12px 600）；② 底部操作栏「加入购物车」「立即购买」**统一合并为一个灰色不可点胶囊**「已售罄」（`#E0E0E0` 底 + `#999` 文字，`opacity .6`），客服按钮保留可点；③ 数量步进器隐藏或整行置灰不可操作；④ 规格选择仍可查看但不可触发加购
