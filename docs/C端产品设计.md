## 2. 商品

- **商品字段**：名称、主图、售价、规格（最多 4 个）、标签（热销 / 新品 / 礼盒 / 特惠）、是否推荐、推荐语、库存、分类、详情图、产品亮点
- 价格**只展示售价（现价）**，不使用划线价；券为额外优惠
- 多规格：不同规格可对应不同价格；详情页默认选中第一个规格，切换规格价格区实时刷新
- **全局不做**：商品评价、商品收藏
- 库存为 0 时标记售罄，详情页购买按钮置灰

## 4. 优惠券

两种券：**满减券**（如满 100 减 10）、**无门槛券**（如直减 5 元）。

- **后台创建字段**：券名、券类型、优惠金额（满减券额外填门槛金额）、有效期、每人限领张数、发行总量
- **「新人」定义**：注册 / 登录后**且无历史已支付订单**的用户（而非「首次进入小程序」），避免「领券不下单」的羊毛账户占用名额与库存
- **感知与展示分工（标签展示完整券名，明确优惠内容）**：
  - 无门槛券（新人）→ 注册后**弹窗领取**，主打拉新；标签 / 入口**展示券名「5元券」**，让用户明确优惠
  - 满减券 → 商品卡片（首页 / 列表）**图片右上角**红底白字标签，**展示完整券名「满100减10」**（让用户明确优惠内容，避免点进去才发现门槛的落差）
  - 均不使用划线价
- **领取入口**：
  1. 新用户注册后首次进首页弹窗领新人券（无门槛券唯一领取入口）
  2. 商品详情页展示可用满减券，可手动领取（已领显示「已领取」）
  3. 进入购物车时系统**自动领取**可用券，无需手动操作
- **发券场景**：新人注册自动发券（弹窗）、分享链接领券、后台手动发券、购物车自动领券
- **结算抵扣（叠加规则）**：无门槛券 + 满减券可**同时生效**，每种类型各用一张（满减券需满足门槛金额）；系统默认选中抵扣金额最大的组合
  - **透明度**：购物车「领券结算」横条**动态展示已用券明细**，如「已自动使用【新人5元券】+【满100减10】，共省 ¥15」，让用户清楚优惠来源，避免出现看不懂的金额
  - **凑单提示**：若用户还有**更高门槛、抵扣更大**的可用券未达门槛，领券横条动态展示引导小字，如「再凑 ¥80 可用满200减30券，多省 ¥20」，把「为什么没提醒我」的潜在抱怨转化为「继续逛逛」的引导
  - **风险控制**：后台提供开关「**购物车自动最优凑单**」——默认**关闭**（商家手动控制风险），开启后系统才自动凑单 / 选最优券组合（详见《后台管理产品设计.md》营销配置）
- **券状态**：未开始 / 进行中 / 已结束 / 已停发
- **「我的优惠券页」**：按 可用 / 已使用 / 已过期 三态分 Tab 展示；可用券彩色票券卡，失效券灰度处理

## 5. 积分

| 规则项 | 设置 |
|--------|------|
| 发放时机 | 用户**确认收货后**才发放（防止先用积分再退款） |
| 消费送积分 | 每消费 1 元 = 1 积分（会员加速见会员表） |
| 拉新送积分 | 邀请好友成功下单，送 20 积分 |
| 积分有效期 | 12 个月 |
| 积分抵现 | 100 积分 = 1 元，最高可抵订单金额 50% |

- 确认订单页提供「积分抵扣」开关；**仅当可抵金额 ≥ ¥0.1 时默认开启**并按可抵额度抵扣，低于阈值则默认关闭并置灰（避免出现「-¥0.00」的无效抵扣视觉干扰）
- **「积分中心页」**：金色头部展示当前积分、可抵金额、会员加速标签；下方积分流水（获取为红色 +、使用为灰色 −），按 全部 / 获取 / 使用 分 Tab；规则说明可查看
- **风控（关单 / 退款处理）**：
  - **未支付取消**：订单用了积分但未支付即取消（含超时自动取消）→ 占用的积分**立即原路退回**
  - **支付后整单退款**：现金原路退回，**抵扣的积分一并返还**；若积分已发放（确认收货后）再退款则同步回收，避免「先用积分再退款」套利
  - **已完成后再申请退货退款**：此时积分已发放，售后审核通过后**同步扣回已发放积分**；若积分余额不足，允许账户暂为负，或从退款金额中按规则（100 积分 = 1 元）抵扣等额现金后再退款
  - 发放时机后置（确认收货后）+ 抵扣占用即时释放，确保正常链路积分不为负

## 7. 订单

### 7.1 用户主链路
加购 → 购物车 → 确认订单（选地址 / 选券 / 积分）→ 微信支付 → 支付结果 → 订单详情

### 7.2 订单状态机

```
待付款 ──支付成功──▶ 待发货 ──后台发货──▶ 待收货 ──确认收货──▶ 已完成
  │                    │                    │
  │超时30min/手动取消    │仅退款(无需寄回)     │退货退款(需寄回)
  ▼                    └─────────┬──────────┘
已取消                          ▼
              退款售后：提交申请 → 商家审核 →[买家寄回 → 商家收货]→ 退款完成
                                （[ ] 内节点仅退货退款需要）
```

| 状态 | 触发 | C 端可执行操作 |
|------|------|--------------|
| 待付款 | 提交订单 | 去支付 / 取消订单 |
| 待发货 | 支付成功 | 申请退款 / 提醒发货 |
| 待收货（配送中） | 后台发货 | 查看物流 / 确认收货 / 申请退货退款 |
| 已完成 | 确认收货 | 再次购买 / 删除订单 |
| 退款售后 | 待发货申请退款 / 待收货申请退货退款 | 查看售后进度 / 填写寄回单号 |
| 已取消 | 超时 30 分钟未支付 / 手动取消 | 删除订单 |

> 个人中心 5 个状态图标与订单列表 6 个 Tab 的「退款售后」为聚合筛选维度。

### 7.3 关键规则
- 待付款订单超过 **30 分钟**未支付自动取消
- **退款 / 售后**（按是否发货分两类）：
  - **仅退款（未发货）**：「待发货」订单可申请，商家审核通过后**直接原路退款**，无需寄回
  - **退货退款（已发货）**：「待收货」订单可申请，需「**提交申请 → 商家审核 → 买家寄回 → 商家确认收货 → 退款**」，**商家收货后才打款**，避免货财两空
  - 退款资金 / 权益：现金原路退回，抵扣积分返还、已发放积分回收，未过期券原路退回（见 5. 积分风控）
- **物流**：发货后订单详情展示物流时间轴（快递节点 + 时间），数据来源后台填写的物流单号对接快递查询；待发货状态不展示物流
- 确认收货后：订单转「已完成」并触发积分发放（见积分规则）
- **金额明细**统一口径：商品合计 / 会员折扣 / 新人券 / 满减券 / 积分抵扣 / 运费 / 实付金额

### 7.4 价格计算规则（会员 / 多券 / 积分叠加）

当一笔订单同时涉及会员折扣、多张优惠券、积分抵扣时，按以下**固定优先级自上而下依次作用**，每一步在上一步结果基础上扣减：

1. **商品合计** = Σ（所选规格售价 × 数量）。以各商品所选规格的售价为准，不使用划线价。
2. **会员折扣**（按会员等级）= 商品合计 ×（1 − 折扣率）。白银 9.8 折、黄金 9.5 折、普通无折扣；金额单列（金色）。
   - 会员价小计 = 商品合计 − 会员折扣
3. **优惠券抵扣**（在会员价小计基础上扣减；无门槛券 + 满减券**各用一张叠加**）：
   - **满减券门槛以「商品合计（原价）」判断**（更利于转化），满足门槛后按券面额直减
   - **无门槛券**直接按面额直减
   - 券抵扣合计 = 无门槛券面额 + 满减券面额
   - 券后小计 = 会员价小计 − 券抵扣（不低于 0）
4. **积分抵扣**（最后扣减）：
   - 抵扣上限 = min（用户可用积分可兑换金额，**券后小计 × 50%**）
   - 100 积分 = 1 元；积分抵扣**不参与运费**
   - 抵扣后小计 = 券后小计 − 积分抵扣
5. **运费**：单独计算，**不参与会员折扣 / 券 / 积分抵扣**。满额包邮门槛以「商品合计」判断，达门槛运费为 0。
6. **实付金额** = 商品合计 − 会员折扣 − 券抵扣 − 积分抵扣 + 运费

**计算示例（确认订单页口径）：**

| 项目 | 金额 | 说明 |
|------|------|------|
| 商品合计 | ¥157.00 | 规格售价 × 数量之和 |
| 会员折扣（白银 9.8 折） | −¥3.14 | 157 × 2% |
| 新人券（无门槛） | −¥5.00 | 直减 |
| 满减券（满 100 减 10） | −¥10.00 | 商品合计 157 ≥ 100，满足门槛 |
| 积分抵扣 | −¥5.00 | 500 积分；上限 = 券后小计 138.86 × 50% = 69.43，未超限 |
| 运费 | ¥0.00 | 商品合计达包邮门槛 |
| **实付金额** | **¥133.86** | 157 − 3.14 − 5 − 10 − 5 + 0 |

**边界与风控：**
- 每一步扣减后金额**均不为负**；会员折扣 + 券 + 积分抵扣总额不超过商品合计
- 同类型券每单各限一张（无门槛 1 张 + 满减 1 张）；系统默认选中**抵扣最大的可用组合**
- 实付金额最低 ¥0.01（微信支付要求）
- 积分在**确认收货后**发放，下单时仅做抵扣占用；**发生退款时按比例回收已抵扣积分与优惠**，避免「先用积分再退款」套利
- 退款时：抵扣的积分原数返还、优惠券若未过期原路退回

## 10. 页面交互逻辑

### 10.1 导航与 Tab Bar
- 底部 Tab Bar 4 个入口：首页 / 发现 / 购物车 / 个人中心
- 商品详情、确认订单、支付结果、订单详情、订单列表、我的优惠券、积分中心**不显示** Tab Bar
- 加购操作后底部购物车图标角标 +1

### 10.2 首页
- App 启动默认页；顶部仅 Logo + 品牌名（无搜索框、无消息图标）
- **Banner**：来源后台「内容管理-Banner」，点击跳转商品列表页；纯图轮播
- **新人券弹窗**：新用户注册后首次进首页自动弹出，领取无门槛新人券
  - **异常兜底**：① 领券接口失败 / 网络异常时，弹窗**原地提示**「网络开小差了，再试一次」并保留重试按钮，**不自动关闭**；② 用户**未领取就关闭**（点遮罩 / ✕）→ 下次再进首页**仍会再次弹出**，直至成功领取或券已失效；③ 后台对该新人券维持「**待领取**」临时占位状态，领取成功才正式入账，防止超发漏洞
- **品类专区**：来源后台「分类管理」，点击跳转商品列表页并传分类 ID
- **热销爆款**：按销量 Top N 横向滑动，「查看全部」进商品列表
- **精选推荐**：后台「推荐」商品，左右图文卡突出推荐语；无推荐则按上架时间倒序展示全部
- 商品卡片价格旁「券」标签：该商品有进行中的可用满减券时展示，否则不展示

### 10.3 商品列表页（＝发现）
- 入口：Tab「发现」（全部商品）/ 首页 Banner、品类、热销「查看全部」（带返回的子页）
- 顶部仅返回 + 品类名（无搜索框、无排序栏）
- 从首页品类进入默认选中对应品类；仅品类筛选 + 双列瀑布流
- 无结果进入空状态：图标 +「这里还没有好物～」+「随便逛逛」

### 10.4 商品详情页
- 价格只展示售价；下方展示会员价，非会员展示开通会员引导
- 规格选择：点击选中高亮，价格实时切换；默认选第一个规格
- 数量步进：最小值 1，为 1 时减量无效
- 加入购物车：留在当前页，购物车角标 +1
- 立即购买：进确认订单页
- 领券：展示可用满减券，点击领取，已领显示「已领取」
- 底部栏：客服 + 加入购物车 + 立即购买（无评价、无收藏）

### 10.5 购物车页
- 入口：Tab「购物车」
- 领券结算横条：进入自动领取并匹配最优券（抵扣最大），无需手动；展示「已自动领取最优券，可省 ¥X」；无可用券隐藏
- 勾选框：单项选中 / 取消，底部合计实时更新
- 数量最小值 1；勾选状态影响合计与结算数量
- 删除最后一件进空状态：空车图标 +「购物车还是空的」+「去逛逛」跳首页
- 结算按钮 N = 已勾选件数；无选中不触发结算

### 10.6 确认订单页
- 无地址显示「请添加收货地址」，点击进新增地址页；地址为空时提交需提示先填地址
- 商品信息仅展示，不可改数量（返回购物车修改）
- 优惠券：默认选中抵扣最大组合，不满足门槛的券置灰
- 积分抵扣：开关默认开启，展示可抵额度
- 点击提交调起微信支付

### 10.7 支付结果页
- 微信支付回调后自动跳转
- 成功：展示实付、已省、确认收货后可得积分；操作「查看订单」/「返回首页」
- 失败：展示失败状态及「重新支付」按钮

### 10.8 订单详情页
- 状态流转：待付款 → 待发货 → 待收货（配送中）→ 已完成（另有退款售后 / 已取消）
- 物流信息：**不再常驻页面**，发货后通过底部「查看物流」按钮（或状态横幅最新物流摘要）触发**弹窗**展示物流时间轴（最新节点在最上）；待发货状态无入口
  - **弹窗轻量化**：物流节点**超过 5 条时，弹窗默认只展示最新 3-4 条**，底部提供「查看全部物流 ›」入口，跳转**全屏物流详情页**展示完整节点，保持弹窗轻量
- **售后进度条**：订单进入退款 / 售后流程时，物流时间轴位置**切换为售后进度条**，节点清晰：提交申请 → 商家审核 →（退货退款）买家寄回 → 商家收货 → 退款完成；当前节点高亮，退货退款时展示「填写寄回单号」入口
- 金额明细：商品合计 / 会员折扣 / 新人券 / 满减券 / 积分抵扣 / 运费 / 实付金额
- 未完成订单展示「确认收货后可得 N 积分」
- 底部操作按状态变化：待付款（去支付 + 取消）/ 待发货（申请退款）/ 待收货（确认收货 + 查看物流 + 申请退货退款）/ 已完成（再次购买 + 删除）/ 退款售后（查看进度）
- **已完成订单保留「申请售后」入口**：已完成状态底部操作区 / 页面底部保留一个「申请售后」文字链接，点击引导至客服会话或简化售后申请流程（在订单列表「退款售后」Tab 之外的必要入口，避免确认收货后发现问题却找不到入口）

### 10.9 订单列表页
- 入口：个人中心「我的订单」5 个状态图标或「全部 ›」，进入时自动定位对应状态 Tab
- 顶部：返回 +「我的订单」
- 状态 Tab（吸顶横滑）：全部 / 待付款 / 待发货 / 待收货 / 退款售后 / 已完成
- 订单卡片：订单号·状态 + 商品行（可多件，「共 N 件」）+ 实付金额 + 按状态变化的操作按钮
- **操作按钮按状态**（与详情页一致，列表页直接露出常用操作）：待付款（去支付 + 取消）/ 待发货（提醒发货）/ **待收货（确认收货 + 查看物流 + 申请退货退款）**/ 已完成（再次购买 + 删除）/ 退款售后（查看进度）。「申请退货退款」在列表页直接露出，减少用户路径
- 点击订单卡进订单详情页；无数据进空状态

### 10.10 个人中心页
- **会员头部卡**：头像 + 昵称 + 会员徽章 + 升级进度 + 权益（折扣 / 积分加速），合并为一张金色卡
- **资产入口**（两等分）：优惠券（N 张可用）/ 积分（N），点击分别进我的优惠券页 / 积分中心页（无收藏）
- **我的订单**：① 5 个状态图标入口（待付款 / 待发货 / 待收货 / 退款售后 / 已完成），点击进订单列表页对应 Tab；② 下方仅展示**最新 1 笔订单摘要**（商品图 + 名称 + 状态 + 实付价 + **右侧 › 箭头**），**整行为点击热区**，点击进订单详情页；无订单隐藏（方案 A：最大化节省空间，避免功能菜单入口被挤到过低）
- **功能菜单**（3 项）：地址管理 / 联系客服 / 关于我们；优惠券、积分中心不再在菜单重复（统一走资产入口），已去除「我的收藏」「设置」
- 「关于我们」进品牌故事页，数据来源后台「内容管理」

### 10.11 我的优惠券页
- 入口：个人中心资产入口「优惠券」
- 状态 Tab：可用 / 已使用 / 已过期，「可用」带数量
- 券卡：横向票券造型，左金额区（无门槛 / 满减门槛）+ 右券名与有效期 +「去使用」按钮
- **临期倒计时**：可用券有效期**不足 24 小时**时，有效期文字旁展示**红色动态倒计时**「· 仅剩 5时30分」制造紧迫感，优于仅展示过期日期
- 失效券灰度处理；无可用券进空状态

### 10.12 积分中心页
- 入口：个人中心资产入口「积分」
- 金色头部：当前积分大数字 + 可抵金额（当前可抵 ¥X）+ 会员加速标签 + 「规则 ›」
- 明细 Tab：全部 / 获取 / 使用
- 积分流水：标题 + 时间 + 积分变动（获取红 +、使用灰 −）
- 「规则 ›」展示积分规则说明（100 积分 = 1 元、确认收货发放、会员加速、最高抵 50%）
