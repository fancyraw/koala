## 2. 商品

- **商品字段**：名称、主图、售价、规格（最多 4 个）、标签（热销 / 新品 / 礼盒 / 特惠）、是否推荐、库存、分类、详情图、产品亮点
- 价格**只展示售价（现价）**，不使用划线价；券为额外优惠
- 多规格：不同规格可对应不同价格；详情页默认选中第一个规格，切换规格价格区实时刷新；**单规格商品规格区可不展示（或展示但不可切换）**
- **全局不做**：商品评价、商品收藏
- 库存为 0 时标记售罄，详情页购买按钮置灰
- **全部商品统一参与营销**：无单品营销开关，所有商品默认参与满减券与无门槛券；C 端的**券优惠、到手价计算**对全部商品统一生效

### 2.1 商品卡片（通用组件）

商品卡片在首页热销爆款、首页精选推荐、商品列表瀑布流三处复用，统一信息结构：**商品图 + 商品名 + 价格 + 加购按钮**。

- **不展示优惠券标签**（券优惠统一在购物车 / 结算页体现，卡片只露价格，保持干净种草感）
- **商品标签角标**：商品图左上角展示**后台配置的单个商品标签**（热销 / 新品 / 礼盒 / 特惠，红标贴角）。后台未配标签则不展示；**不再有按销量自动计算的「爆款」角标**
- **不展示「已售 N」**，原位置改为**加购按钮**（28px 圆形品牌红 + 白色「+」图标）
- **加购按钮交互**：单规格商品点击直接加入购物车；多规格商品点击弹出底部「选择规格」浮层（选规格 + 数量后加入）。加入成功购物车角标 +1 并 Toast「已加入购物车」；售罄商品按钮置灰
- 点击卡片其余区域进商品详情页

| 模块 | 取数逻辑 | 展示形态 |
|------|---------|---------|
| 热销爆款 | 按销量排序取 Top N | 形态 A 横向滑动竖卡，左上展示后台配置的商品标签角标（无则不展示） |
| 精选推荐 | 后台标记「推荐」的商品；无推荐时按上架时间倒序展示全部 | 形态 B 单列左右图文卡（图左信息右，突出种草语＝复用商品「产品亮点」首条，高屏效） |
| 商品列表 | 按分类 / 搜索过滤 | 形态 C 双列瀑布流卡 |

## 4. 优惠券

两种券：**满减券**（如满 100 减 10）、**无门槛券**（如直减 5 元）。

- **后台创建字段**：券名、券类型、优惠金额（满减券额外填门槛金额）、有效期、每人限领张数、发行总量、**适用用户标签**
- **用户标签（下发资格）**：券模版可绑定「适用用户标签」限制下发对象，标签由系统**实时函数计算**（不选＝通投全员）。「新人」即其中一个标签
- **「新人」定义**：注册 / 登录后**且无历史已支付订单**的用户（而非「首次进入小程序」），避免「领券不下单」的羊毛账户占用名额与库存
- **券的来源（系统下发，无手动领取 = 后端自动领）**：用户登录 / 进首页时前端调一次「查询可领券」接口，后端按券模版的用户标签资格实时判定并**自动领取**生成用户券，前端不做「领取」动作。**资格只在领取时刻校验，券到账后即为用户资产，后续标签变化（如下首单后不再是新人）不影响已到账券**
  1. **标签通投券（含新人券 / 活动券）**：后端按券模版绑定的用户标签自动下发——配「新人」标签即新人券，不配标签即全员活动券
- **感知与展示分工（标签展示完整券名，明确优惠内容）**：
  - 商品卡片（首页 / 列表）**不再展示券标签**，券优惠统一在购物车 / 结算页体现
  - 商品详情页**不展示券**，券优惠统一在购物车 / 结算页体现
  - 均不使用划线价
- **统一领券弹窗（系统下发，前端查到即弹）**：
  1. **券由系统 / 后台下发到用户账户**，前端不做"领取"动作
  2. 前端在用户登录 / 进首页时**查询账户是否有新到账、未告知的券**，**查到则弹窗提示**（无新券则不弹）
  3. 弹窗**统一展示新到账的券列表（支持多张）**，告知「已为你发放 N 张券」
  4. 按钮为「**去使用**」（券已在账户，引导去购物），而非「立即领取」
  5. 异常兜底：查询接口失败本次不弹、下次再查；用户未操作关闭则下次进首页若仍有未告知券会再次提示
- **结算抵扣（叠加规则）**：无门槛券 + 满减券可**同时生效**，每种类型各用一张（满减券需满足门槛金额）；系统默认选中抵扣金额最大的组合
  - **透明度**：购物车底部结算栏价格区**直接体现已用券**，如「已省 ¥15」；用券规则为系统自动最优，用户**只能查看当前选中的券、不可手动切换**
  - **凑单提示**：当存在因未达门槛而无法使用的更大额券时，在结算栏上方**独立卡片**提示「再买 ¥43 可省 ¥20」，引导用户凑单解锁更多优惠
  - **自动最优券**：下单时系统**始终**自动选中抵扣金额最大的可用券组合（无门槛 + 满减各一张），无开关、无手动切换
- **券状态**：未开始 / 进行中 / 已结束 / 已停发
- **「我的优惠券页」**：按 可用 / 已使用 / 已过期 三态分 Tab 展示；可用券彩色票券卡，失效券灰度处理

## 5. 订单

### 5.1 用户主链路
加购 → 购物车 → 确认订单（选地址 / 选券）→ 微信支付 → 支付结果 → 订单详情

### 5.2 订单状态机

```
待付款 ──支付成功──▶ 待发货 ──后台发货──▶ 待收货 ──确认收货──▶ 已完成
  │                    │                    │
  │超时30min/手动取消    │仅退款(无需寄回)     │退货退款(需寄回)
  ▼                    └─────────┬──────────┘
已取消                          ▼
              退款售后：提交申请 → 商家审核 →[买家寄回 → 商家收货]→ 退款完成
                                （[ ] 内节点仅退货退款需要）
```

| 状态 | 触发 | C 端可执行操作 |
|------|------|--------------|
| 待付款 | 提交订单 | 去支付 / 取消订单 |
| 待发货 | 支付成功 | 申请退款 / 联系客服 |
| 待收货（配送中） | 后台发货 | 查看物流 / 申请售后 / 确认收货 |
| 已完成 | 确认收货 | 再次购买 / 删除订单 |
| 退款售后 | 待发货申请退款 / 待收货申请售后 | 查看售后进度 / 填写寄回单号 |
| 已取消 | 超时 30 分钟未支付 / 手动取消 | 删除订单 |

> 个人中心 5 个状态图标与订单列表 6 个 Tab 的「退款售后」为聚合筛选维度。

### 5.3 售后单状态机

一张售后单独立于订单状态流转，按申请类型走不同节点：

```
仅退款（未发货）：  待审核 ──商家通过──▶ 退款成功
                    └──商家驳回──▶ 退款关闭（订单回「待发货」）

退货退款（已发货）：待审核 ──通过──▶ 待买家寄回 ──填单号──▶ 待商家收货 ──确认──▶ 退款成功
                    └──驳回──▶ 退款关闭（订单回「待收货」）
                    ＊买家仅可在「待审核」阶段主动「撤销申请」→ 退款关闭；审核通过进入寄回流程后不可撤销
```

| 售后单状态 | 含义 | 进入条件 | 后续 |
|-----------|------|---------|------|
| **待审核** | 已提交、等待商家处理 | 提交申请 | 通过 / 驳回；买家可撤销 |
| **待买家寄回** | 退货退款审核通过，待买家寄回并填单号 | 退货退款审核通过 | 填写寄回单号 → 待商家收货 |
| **待商家收货** | 买家已寄回，等待商家签收 | 填写寄回单号 | 商家确认收货 → 退款成功 |
| **退款成功** | 款项原路退回完成（终态） | 仅退款审核通过即达 / 退货退款商家收货后达 | — |
| **退款关闭** | 商家驳回或买家撤销（终态） | 审核驳回 / 买家撤销 | 订单回到原状态，可重新申请 |

> **仅退款** 无「待买家寄回 / 待商家收货」两节点，审核通过即「退款成功」（通过即打款）；**退货退款** 商家确认收货后才打款，避免货财两空。
> **驳回 / 撤销后** 订单回到原状态（仅退款→待发货 / 退货退款→待收货），可在有效期内重新发起售后。
> 各售后单状态的**页面展示**（订单列表 / 售后进度页）见 §8.8.3.1。

### 5.4 关键规则
- 待付款订单超过 **30 分钟**未支付自动取消
- **退款 / 售后**（按是否发货分两类）：
  - **仅退款（未发货）**：「待发货」订单可申请，商家审核通过后**直接原路退款**，无需寄回
  - **退货退款（已发货）**：「待收货」订单可申请，需「**提交申请 → 商家审核 → 买家寄回 → 商家确认收货 → 退款**」，**商家收货后才打款**，避免货财两空
  - 退款资金 / 权益：现金原路退回，未过期券原路退回
- **物流**：发货后订单详情展示物流时间轴（快递节点 + 时间），数据来源后台填写的物流单号对接快递查询；待发货状态不展示物流
- **自动确认收货**：「待收货」订单发货后 **7 天**未手动确认收货，系统自动转「已完成」
- 确认收货后：订单转「已完成」；**已完成订单不再展示「查看物流」入口**
- **金额明细**统一口径：商品合计 / 新人券 / 满减券 / 运费 / 实付金额（**无积分抵扣**）

### 5.5 价格计算规则（多券叠加）

当一笔订单同时涉及多张优惠券时，按以下**固定优先级自上而下依次作用**，每一步在上一步结果基础上扣减：

1. **商品合计** = Σ（所选规格售价 × 数量）。以各商品所选规格的售价为准，不使用划线价。
2. **优惠券抵扣**（无门槛券 + 满减券**各用一张叠加**）：
   - **满减券门槛以「商品合计（原价）」判断**（更利于转化），满足门槛后按券面额直减
   - **无门槛券**直接按面额直减
   - 券抵扣合计 = 无门槛券面额 + 满减券面额
   - 券后小计 = 商品合计 − 券抵扣（不低于 0）
3. **运费**：单独计算，**不参与券抵扣**。满额包邮门槛以「商品合计」判断，达门槛运费为 0。
4. **实付金额** = 商品合计 − 券抵扣 + 运费

**计算示例（确认订单页口径）：**

| 项目 | 金额 | 说明 |
|------|------|------|
| 商品合计 | ¥157.00 | 规格售价 × 数量之和 |
| 新人券（无门槛） | −¥5.00 | 直减 |
| 满减券（满 100 减 10） | −¥10.00 | 商品合计 157 ≥ 100，满足门槛 |
| 运费 | ¥0.00 | 商品合计达包邮门槛 |
| **实付金额** | **¥142.00** | 157 − 5 − 10 + 0 |

**边界与风控：**
- 每一步扣减后金额**均不为负**；券抵扣总额不超过商品合计
- 同类型券每单各限一张（无门槛 1 张 + 满减 1 张）；系统默认选中**抵扣最大的可用组合**
- 实付金额最低 ¥0.01（微信支付要求）
- 退款时：优惠券若未过期原路退回

## 8. 页面交互逻辑

### 8.1 导航与 Tab Bar
- 底部 Tab Bar 4 个入口：首页 / 分类 / 购物车 / 个人中心
- 商品详情、确认订单、支付结果、订单详情、订单列表、我的优惠券**不显示** Tab Bar
- 加购操作后底部购物车图标角标 +1
- **购物车角标实时更新**：加购 +1、购物车内删除商品或修改数量时角标同步刷新，角标数 = 购物车内所有商品件数之和；超过 99 显示「99+」，为 0 时隐藏角标
- **首页右上角购物车入口**：首页顶部右上角放购物车图标（带件数角标），点击进购物车页

### 8.1.1 未登录态全局规则
- **可浏览不可操作**：首页、商品列表页、商品详情页**允许未登录浏览**（商品信息、价格正常展示）
- **操作拦截跳登录**：以下操作在未登录时**统一拦截**，弹出微信授权登录弹窗，登录成功后自动继续原操作：
  - 加入购物车 / 立即购买
  - 进入购物车页（Tab / 首页购物车入口点击）
  - 进入个人中心页（Tab 点击）
  - 进入确认订单页
- **领券弹窗**：未登录时**不触发**（登录后查询到新到账券才弹）
- **个人中心未登录态**：用户头部卡替换为「点击登录」引导卡（头像占位 + 「登录 / 注册」文字），功能菜单「我的优惠券」数字显示「--」，订单区域隐藏

### 8.2 首页
- App 启动默认页；顶部左侧 Logo + 品牌名，**右上角购物车入口**（带件数角标）；无搜索框、无消息图标
- **Banner**：来源后台「内容管理-Banner」，点击跳转商品列表页；纯图轮播
- **领券弹窗（系统下发，前端查到即弹）**：券由系统 / 后台下发到账户，前端在登录 / 进首页时**查询账户是否有新到账、未告知的券**，查到则弹窗提示新到账券列表（多张），按钮「去使用」（券已到账）
  - **异常兜底**：① 查询接口失败本次不弹、下次再查；② 用户**未操作就关闭**（点遮罩 / ✕）→ 下次再进首页若仍有未告知券**仍会再次提示**
- **品类专区**：来源后台「分类管理」，点击跳转商品列表页并传分类 ID
- **热销爆款**：按销量 Top N 横向滑动（商品卡片·形态 A），「查看全部」进商品列表；卡片不展示券标签 / 已售，图左上角展示后台配置的商品标签角标（无则不展示），价格行右侧为加购按钮
- **精选推荐**：后台「推荐」商品，左右图文卡突出种草语（＝复用商品「产品亮点」首条，无亮点则不展示该行）（商品卡片·形态 B）；无推荐则按上架时间倒序展示全部；卡片不展示券标签 / 已售，图左上角展示后台配置的商品标签角标（无则不展示），价格行右侧为加购按钮

### 8.3 商品列表页（＝分类）
- 入口：Tab「分类」（全部商品）/ 首页 Banner、品类、热销「查看全部」（带返回的子页）
- **顶部：返回 + 搜索框**——输入文本后**按商品名称模糊匹配**实时过滤瀑布流；清空恢复当前品类全部商品
- 从首页品类进入默认选中对应品类；品类筛选 + 双列瀑布流（商品卡片·形态 C）
- 卡片不展示券标签 / 已售，图左上角展示后台配置的商品标签角标（无则不展示），价格行右侧为加购按钮
- 无结果进入空状态：图标 +「没有找到相关商品」/「这里还没有好物～」+「随便逛逛」

### 8.4 商品详情页
- **售价行**：展示**售价**（不划线）+ 右侧已售
- **不展示优惠明细行**：优惠券等到手优惠**统一在购物车页与确认订单页展示**，详情页只露售价
- 规格选择：点击选中高亮，价格实时切换；默认选第一个规格；**单规格商品规格区可不展示或不可切换**
- 数量步进：最小值 1，为 1 时减量无效；**数量达库存上限**时加量无效并 Toast「已达库存上限」；**达单次限购上限**时加量无效并 Toast「该商品每单限购 N 件」
- 加入购物车：留在当前页，购物车角标 +1
- 立即购买：进确认订单页
- 底部栏：客服 + 加入购物车 + 立即购买（无评价、无收藏、无优惠领取入口）

### 8.5 购物车页
- 入口：Tab「购物车」/ 首页购物车入口
- **顶部「我的优惠券」按钮**：购物车页顶部展示「我的优惠券」入口，点击弹窗展示当前账户**所有优惠券**（可用 / 已用 / 已过期分组，仅查看，不在此处选券）
- **优惠信息浓缩到底部结算栏（去掉领券横条）**：进入购物车系统仍**后台自动匹配最优券**，前端不再用横条提示，改为在价格区直接体现
- **底部结算栏（两行）**：第一行「已省 ¥15」（红色）+「合计 ¥142.00」（真实到手价，黑色加粗）+「结算(N)」；第二行小字「**运费 ¥0.00（满 99 包邮）**」+「原价 ¥157.00」作为参照锚点。未达包邮门槛展示「运费 ¥8.00 · 再买 ¥20 包邮」；无优惠时不展示已省 / 原价
- **用券只读**：券由系统自动选中抵扣最大组合，**用户不可手动切换**；点击「已省 ¥15」可查看当前已用券明细（仅查看）
- **凑单提示卡片**：当存在因未达门槛而无法使用的更大额满减券时，在结算栏**上方独立卡片**提示「再买 ¥43 可省 ¥20」，点击可跳商品列表凑单；无可凑券时不展示该卡片
- 勾选框：单项选中 / 取消，底部合计实时更新
- 数量最小值 1；勾选状态影响合计与结算数量
- 删除最后一件进空状态：空车图标 +「购物车还是空的」+「去逛逛」跳首页
- 结算按钮 N = 已勾选件数；无选中不触发结算
- **多规格商品**：同一商品不同规格拆为独立行，可分别勾选、修改数量、删除
- **商品失效处理**：商品下架 / 库存为 0 时，购物车内对应商品标记「失效」，置灰不可勾选，底部展示「删除失效商品」文字按钮；失效商品不参与合计计算

### 8.6 确认订单页
- 无地址显示「请添加收货地址」，点击进新增地址页；地址为空时提交需提示先填地址
- 商品信息仅展示，不可改数量（返回购物车修改）
- 优惠券：默认选中抵扣最大组合，不满足门槛的券置灰
- **订单备注**：选填文本框，用户可填写配送要求等备注信息，最多 100 字
- **金额明细**：商品合计 / 新人券 / 满减券 / **运费** / 实付金额（**无积分抵扣**）
- **地址切换运费重算**：切换收货地址后运费根据新地址实时重算（不同地区运费可能不同），包邮门槛判断同步刷新，金额明细实时更新
- 点击提交调起微信支付
- **提交前最后校验（防停留过久失效）**：点击「提交订单」时前端再做一次校验，若优惠券已过期 / 商品库存不足，Toast 提示「优惠券已失效」或「『商品名』库存不足」并刷新页面金额与可用券，校验通过才调起支付

### 8.7 支付结果页
- 微信支付回调后自动跳转
- **成功**：展示实付、已省；操作「查看订单」/「返回首页」
- **失败**：展示失败状态及「重新支付」按钮
- **支付中（中间态）**：微信侧延迟 / 银行处理中时展示加载动效 +「支付确认中…」文案 + 「查看订单」按钮（用户可主动离开去订单详情查看最终状态）；前端轮询 / 回调到账后自动切换为成功/失败

### 8.8 订单详情页
- 状态流转：待付款 → 待发货 → 待收货（配送中）→ 已完成（另有退款售后 / 已取消）
- 物流信息：「待收货（配送中）」状态下，订单详情**顶部状态横幅下方展示一条最新物流摘要**（最新节点 + 时间），右侧「查看全部 ›」跳转**独立物流详情页**展示完整时间轴；**待发货状态无物流摘要、已完成订单也不再展示物流入口**。订单列表页「查看物流」按钮直接跳物流详情页
- **售后进度条**：订单进入退款 / 售后流程时，物流时间轴位置**切换为售后进度条**，节点清晰：提交申请 → 商家审核 →（退货退款）买家寄回 → 商家收货 → 退款完成；当前节点高亮，退货退款时展示「填写寄回单号」入口
- 金额明细：商品合计 / 新人券 / 满减券 / 运费 / 实付金额（**无积分抵扣**）
- 底部操作按状态变化：待付款（去支付 + 取消）/ 待发货（申请退款 + 联系客服）/ 待收货（申请售后 + 确认收货，**查看物流入口在顶部物流摘要条、不在底部**）/ 已完成（再次购买 + 删除）/ 退款售后（查看进度）/ 已取消（删除订单）
- **待付款倒计时**：待付款状态横幅展示「剩余 XX:XX 自动关闭」实时倒计时，倒计时归零后前端自动刷新页面切换为「已取消」状态；订单列表页待付款卡片同样展示倒计时
- **已完成订单保留「申请售后」入口**：已完成状态底部操作区 / 页面底部保留一个「申请售后」文字链接，点击引导至客服会话或简化售后申请流程（在订单列表「退款售后」Tab 之外的必要入口，避免确认收货后发现问题却找不到入口）
- **「再次购买」行为**：点击后将订单内所有商品（原规格 × 原数量）**批量加入购物车**，然后跳转购物车页。若原规格已下架 / 售罄，Toast 提示「『商品名』已下架，已为你跳过」并仅加入可用商品；若全部不可用则 Toast「商品已下架」不跳转。价格以当前售价为准，不沿用历史价格
- **「删除订单」二次确认**：点击后弹出居中确认弹窗「确定删除该订单？删除后不可恢复」，操作按钮「取消」+「确定删除」（红色），防止误操作

### 8.8.1 售后申请页
- 入口：订单详情页 / 订单列表页「申请退款」（待发货）或「申请售后」（待收货）
- **申请类型**：根据订单状态自动判断——待发货 → 仅退款；待收货 → 退货退款
- **表单字段**：
  - 退款原因（必选，下拉单选）：不想要了 / 拍错了 / 商品质量问题 / 未按时发货 / 其他
  - 退款金额（自动填充实付金额，不可修改）
  - 补充说明（选填，文本框最多 200 字）
  - 上传凭证（选填，最多 3 张图片，用于质量问题等场景）
- 底部「提交申请」按钮，提交后跳转售后进度页
- **已完成订单的「申请售后」**：点击后引导至客服会话（微信客服），由客服人工处理

### 8.8.2 物流详情页
- 入口：① 订单列表页待收货卡片「查看物流」按钮直接跳入；② 订单详情页顶部物流摘要条「查看全部 ›」跳入
- **内容结构**：物流公司 + 运单号（可复制）；商品摘要（商品图 + 商品名，多件展示「等 N 件」+ 订单号）；完整物流时间轴
- **物流时间轴**：展示全部快递节点，**最新节点在最上且高亮**，每节点含描述 + 时间；数据来源后台填写的物流单号对接快递查询
- **空状态**：暂无物流信息时展示占位图标 + 「暂无物流信息」
- 仅「待收货（配送中）」与流程中订单可进入；待发货无物流、已完成订单不再展示物流入口

### 8.8.3 售后进度页（售后详情页）
- 入口：订单详情页 / 订单列表页「查看进度」「查看售后进度」，或售后申请提交后自动进入
- **顶部售后状态横幅**：展示当前售后状态大字（如「商家审核中」「请寄回商品」「退款成功」）+ 说明
- **售后进度条**：节点 提交申请 → 商家审核 →（退货退款）买家寄回 → 商家收货 → 退款完成；**仅退款流程省略「买家寄回」「商家收货」两节点**。已完成节点实心、当前节点高亮、未达节点灰色
- **退货退款阶段**：「买家寄回」节点展示「填写寄回单号」入口，填写后进入「待商家收货」
- **信息区**：商品信息 + 退款金额（=实付金额）+ 售后信息（售后编号 / 申请时间 / 退款原因 / 退款方式：现金原路退回，未过期券一并退回）
- **底部操作按阶段变化**：等待审核（撤销申请 + 联系客服）/ 待寄回（填写单号 + 联系客服）/ 待商家收货（修改单号 + 联系客服）/ 退款完成（返回订单 / 无操作）/ 退款关闭（联系客服）

### 8.9 订单列表页
- 入口：个人中心「我的订单」5 个状态图标或「全部 ›」，进入时自动定位对应状态 Tab
- 顶部：返回 +「我的订单」
- 状态 Tab（吸顶横滑）：全部 / 待付款 / 待发货 / 待收货 / 退款售后 / 已完成
- **各状态说明（卡片状态文案 + 含义）**：
  - **待付款**：已提交未支付，状态文案旁展示倒计时「29:59」，超 30 分钟自动取消转「已取消」
  - **待发货**：已支付，商家备货中
  - **待收货**：商家已发货、配送中（可查看物流）；发货 7 天未确认自动转「已完成」
  - **退款售后**：聚合维度，含「仅退款 / 退货退款」流程中的订单（对应订单详情的退款售后态）
  - **已完成**：已确认收货（含 7 天自动确认收货）
  - **已取消**：超时未付 / 手动取消；**无独立 Tab，归在「全部」Tab 下展示**
- 订单卡片：订单号·状态 + 商品行（可多件，「共 N 件」）+ 实付金额 + 按状态变化的操作按钮
- **操作按钮按状态**（与详情页一致，列表页直接露出常用操作）：待付款（去支付 + 取消）/ 待发货（联系客服 + 申请退款）/ **待收货（查看物流 + 申请售后 + 确认收货）**/ 已完成（再次购买 + 删除）/ 退款售后（查看进度）。「申请售后」在列表页直接露出，减少用户路径
- 点击订单卡进订单详情页；无数据进空状态

### 8.10 个人中心页
- **用户头部卡**：头像 + 昵称 + 「编辑资料 ›」的简洁信息卡（**无会员体系**，不展示等级徽章 / 升级进度 / 权益）；未登录时替换为「点击登录」引导卡
- **我的订单**：① 5 个状态图标入口（待付款 / 待发货 / 待收货 / 退款售后 / 已完成），点击进订单列表页对应 Tab；**不再展示最新订单摘要**（保持模块简洁）
- **功能菜单**（4 项）：地址管理 / **我的优惠券**（紧随地址管理之后，右侧展示可用券数量）/ 联系客服 / 关于我们；**已去除「资产入口」「我的收藏」「设置」「积分中心」**
- 「关于我们」进品牌故事页，数据来源**前端配置文件**

### 8.11 我的优惠券页
- 入口：个人中心功能菜单「我的优惠券」
