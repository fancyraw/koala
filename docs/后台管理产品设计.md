# 后台管理产品设计

## 一、概述

后台供管理员进行商城运营管理，覆盖商品、营销、订单、用户、内容、系统配置与运维等模块。

> C 端产品需求见《C端产品设计.md》；UI 设计见《后台管理UI设计.md》、《后台管理UI原型.html》。

## 二、角色与权限

| 角色 | 可访问范围 |
|------|-----------|
| 普通管理员 admin | 数据看板、商品、订单、用户、优惠券、营销配置、内容、系统设置 |
| 超级管理员 super_admin | 以上全部 + **运维管理**（系统状态 / 操作日志 / 数据操作） |

- 所有写操作记入操作日志（操作人 + 时间 + 类型 + 内容）
- 禁用用户后该用户无法登录商城

## 三、模块清单

| 模块 | 主要职责 |
|------|---------|
| 数据看板 | 核心经营指标与趋势分析（含退款、优惠券核销数据） |
| 商品管理 | 商品列表、分类管理、库存 |
| 订单管理 | 订单查询、发货、物流单号、退款 / 售后处理 |
| 用户管理 | 用户列表、会员等级、用户详情 |
| 优惠券管理 | 券的创建 / 编辑 / 停发与核销数据 |
| 营销配置 | 会员等级与升级礼券、购物车自动凑单开关 |
| 内容管理 | Banner、品牌信息 / 品牌故事 |
| 系统设置 | 配送（运费 / 包邮门槛）、支付配置、订单超时配置 |
| 运维管理 | 系统状态、操作日志、数据操作（仅超级管理员） |

> **说明**：本商城**不设积分体系**；会员权益为「升级时一次性发放专属优惠券」（升级送券），无下单折扣、无积分。

---

## 四、各模块产品需求

### 1. 数据看板

- **入口**：登录后默认首页 / 侧栏「数据看板」
- **时间段切换**：今日 / 本周 / 本月，影响全部指标与图表
- **核心指标卡**（4 张）：今日订单、今日销售额、新增用户、总用户数；含环比（↑绿 / ↓红）
- **第二行指标卡**（4 张，补充运营数据）：
  - 待处理退款：当前待审核的退款申请数量，红色高亮提醒
  - 今日退款额：当日已退款总金额
  - 优惠券核销率：进行中优惠券的整体核销率（已用 / 已领）
  - 今日客单价：当日销售额 / 当日订单数
- **分析 Tab**：
  - 销售趋势：折线图（日期 × 销售额）
  - 热销商品：排行表（商品 / 销量 / 销售额），「查看全部」跳商品管理
  - 订单分布：各状态订单占比饼图
  - 用户分析：新增用户趋势 + 会员等级分布
  - 退款分析：退款趋势（日期 × 退款额）+ 退款原因分布饼图
- 刷新按钮重载全部数据

### 2. 商品管理

#### 2.1 商品列表
- **工具栏**：搜索（商品名）+ 分类筛选 + 状态筛选（上架 / 下架 / 售罄）
- **列表字段**：主图、名称、售价、标签、是否推荐、库存、分类、状态、操作
- **库存预警**：库存 < 20 显示黄色「库存不足」；库存 = 0 显示红色「已售罄」并自动置为售罄状态
- **操作**：新增 / 编辑 / 上架 / 下架；分页
- **商品字段**：名称、主图、售价（只售价无划线价）、规格（最多 4 个，可对应不同价格）、标签（热销 / 新品 / 礼盒 / 特惠，可多选）、**是否推荐（用于 C 端首页精选推荐区）**、库存、分类、详情图、产品亮点
- **营销参与开关（每个商品 2 项，默认全部关闭，商家手动开启）**：
  | 开关 | 开启效果 |
  |------|---------|
  | 参与满减券 | 开启后该商品**可用满减券**（计入满减门槛并被抵扣），否则不参与满减 |
  | 参与无门槛券 | 开启后该商品**可用无门槛券**，否则无门槛券不可用于该商品 |
  > 用于商家精细控制每个商品的优惠成本与利润；C 端的券标签、到手价计算均依据这两个开关。**会员无下单折扣，故无「参与会员折扣」开关**

#### 2.2 分类管理
- **字段**：分类名称、图标、排序、状态
- **操作**：新增 / 编辑 / 启用 / 禁用 / 删除；拖拽 `≡` 调整排序
- **规则**：
  - 同步到 C 端首页品类专区与商品列表筛选栏
  - **仅商品数量为 0 的分类可删除**
  - 禁用的分类 C 端不展示

### 3. 订单管理

- **状态 Tab**（附数量）：全部 / 待付款 / 待发货 / 待收货 / 已完成 / 已取消 / 退款售后
- **搜索**：订单号 / 买家姓名
- **列表**：订单号、买家、商品（首件 + 共 N 件）、金额、状态、下单时间、操作；点击行展开详情（收货人 / 地址 / 商品明细 / 优惠券 / 运费 / 实付 / **订单备注**）
- **导出订单**：导出当前筛选条件下的订单
- **发货**：仅待发货订单可操作，弹窗填写**物流公司 + 物流单号** → 转待收货；单号同步 C 端订单详情物流时间轴
- **退款 / 售后处理**（核心，防货财两空）：
  - **仅退款（未发货）**：待发货订单申请退款，商家审核通过后**直接原路退款**，无需寄回
  - **退货退款（已发货 / 已收货）**：流程「**提交申请 → 商家审核 → 买家寄回 → 商家确认收货 → 退款**」，**商家确认收货后才打款**
  - 「处理退款」支持：同意 / 拒绝、填写退款金额、记录买家寄回单号、确认收货并打款
  - 退款资金与权益：现金原路退回；优惠券若未过期原路退回
- **超时取消**：待付款订单超过 30 分钟未支付自动取消（超时时长可在系统设置中配置）
- **售后处理详细流程**：
  - 列表中退款售后订单展示**售后状态子标签**：待审核 / 已同意·待寄回 / 已寄回·待确认 / 已退款 / 已拒绝
  - **审核操作**：点击「处理退款」弹窗，展示买家申请信息（退款原因 / 退款金额 / 补充说明 / 凭证图片），操作「同意」/「拒绝」（拒绝需填写理由）
  - **仅退款（未发货）**：同意后系统自动原路退款，订单转「已退款」
  - **退货退款（已发货）**：同意后等待买家寄回 → 买家填写寄回单号后订单展示寄回物流信息 → 商家点击「确认收货并退款」→ 系统原路退款
  - **退款资金与权益处理**：现金原路退回；未过期优惠券原路退回
  - **已完成订单售后**：C 端引导至客服，后台可在订单详情中手动发起退款（需填写退款金额与原因）

### 4. 用户管理

- **搜索**：昵称 / 手机号
- **列表**：头像、昵称、手机号、**会员等级**、累计消费、订单数、注册时间、状态、操作
- **会员等级**：由累计消费自动判定——🥉 普通（注册即享）/ 🥈 白银（≥500 元）/ 🥇 黄金（≥1500 元）。**会员权益为升级送券**（无折扣、无积分），升级门槛与礼券可在「营销配置」中调整
- **用户详情页**（点击「详情」进入）：
  - **基本信息**：头像、昵称、手机号、会员等级、累计消费、注册时间、状态
  - **消费记录 Tab**：该用户的订单列表（订单号 / 商品 / 金额 / 状态 / 时间），可点击查看订单详情
  - **优惠券 Tab**：该用户持有的优惠券列表（券名 / 类型 / 来源：新人 / 升级礼券 / 活动 / 状态：可用·已使用·已过期 / 使用时间）
- **禁用**：切换状态，禁用后无法登录商城
- **新人定义**：注册 / 登录后**且无历史已支付订单**的用户（用于新人券发放资格校验）

### 5. 优惠券管理

- **状态 Tab**：全部 / 进行中 / 未开始 / 已结束 / 已停发
- **券卡展示**：券名、类型、有效期、限领、**已领 X/总量**、**已用 + 核销率**、状态、操作
- **新建 / 编辑字段**：券名、券类型（满减 / 无门槛）、优惠金额（满减券额外填门槛金额）、有效期、每人限领张数、发行总量
- **规则**：
  - 进行中的券由系统下发到用户账户（新人券、活动券）；会员升级礼券由营销配置的升级规则下发
  - **已开始的券不可删除，只能停发**
  - 券状态：未开始 / 进行中 / 已结束 / 已停发
- **购物车自动最优凑单开关**：已移至「营销配置」模块统一管理

### 6. 营销配置

> 集中管理会员升级规则与购物车凑单规则，便于商家统一调整营销策略。

#### 6.1 会员等级与升级礼券配置
- **等级列表**（表格展示，可编辑）：

  | 等级 | 升级门槛（累计消费） | 升级礼券（一次性发放） | 当前人数 | 操作 |
  |------|---------------------|----------------------|---------|------|
  | 🥉 普通会员 | 注册即享 | 无 | 180 | — |
  | 🥈 白银会员 | ≥ 500 元 | 5 元无门槛券 ×1 + 满200减20 券 ×1 | 86 | 编辑 |
  | 🥇 黄金会员 | ≥ 1500 元 | 10 元无门槛券 ×2 + 满300减50 券 ×1 | 20 | 编辑 |

- **编辑字段**：升级门槛金额 + **升级礼券配置**（可配置若干张券，每张含：券类型「满减 / 无门槛」、面额、满减门槛、张数）
- **规则**：
  - 等级数量固定 3 级不可增删；普通会员不可编辑（注册即享，无礼券）
  - 修改后**实时生效**，已有用户按新门槛重新判定等级
  - **会员无下单折扣、无积分**；权益完全体现为升级时一次性发放的专属优惠券
  - **升级触发**：累计消费（以支付成功订单累计，退款扣回）达门槛时自动升级并下发对应礼券；同一等级礼券只发放一次
- **会员统计**：各等级用户数量分布（普通 N 人 / 白银 N 人 / 黄金 N 人）

#### 6.2 购物车自动凑单
- **开关**：默认**关闭**
  - 关闭：购物车不自动凑单 / 不自动叠加最优券（仅按用户已有券计算），优惠成本商家手动可控
  - 开启：系统自动选**抵扣金额最大**的券组合
  - 目的：商家初期手动控制优惠力度与利润风险，熟悉后再开启

### 7. 内容管理

#### 7.1 Banner
- **字段**：图片、跳转链接、排序、上下线状态
- **操作**：新增 / 编辑 / 上线 / 下线 / 删除；拖拽排序
- **规则**：同步 C 端首页 Banner 轮播（**纯图，不叠加标签**）；**仅上线状态展示**；下线可删除，删除后序号重排

#### 7.2 品牌信息
- **基本信息**：品牌名称、App 图标、Logo
- **品牌故事**：标题、封面图、纯文本简介（≤200 字）
- **规则**：品牌名称同步 C 端顶部导航与页面标题；品牌故事同步个人中心「关于我们」品牌故事页

### 8. 系统设置

- **配送设置**：基础运费、满额包邮门槛（同步 C 端运费计算与包邮判断）
- **支付设置**：微信支付开关、商户号、API 密钥（脱敏展示）
- **订单设置**：
  - 待付款超时时间：默认 30 分钟，可配置（10~120 分钟）
  - 自动确认收货天数：发货后 N 天自动确认收货，默认 15 天
- 保存成功 Toast，重置恢复上次保存值

### 9. 运维管理（仅超级管理员）

- **系统状态**：服务 / 数据库 / Redis / 磁盘状态 + JVM 内存、CPU、负载监控，自动刷新
- **操作日志**：全部写操作记录（时间 / 操作人 / 类型 / 内容），支持搜索 / 类型筛选 / 日期范围 / 分页
- **数据操作**：
  - SQL 查询（只读）直接执行
  - SQL 修改需**二次确认**（显示 SQL + 影响行数预估）
  - **禁止 DROP/TRUNCATE 等危险语句**；查询结果最多返回 500 行；所有数据操作记入操作日志
